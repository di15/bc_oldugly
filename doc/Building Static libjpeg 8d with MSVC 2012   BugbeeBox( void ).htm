Building Static libjpeg 8d with MSVC 2012

Posted on November 18, 2012 by Jack

This post will explain how to obtain and build the libjpeg C programming library from source, using MS Visual Studio 2012 on Windows 7. The result will be a static release version that you can use in your C/C++ projects for manipulating JPEG images, and as a prerequisite for building other libraries.

Downloads

  • MS Visual Studio 2012 Express
  • The zipped libjpeg 8d source code
  • MS Windows 7 SDK

The Environment

  1. Install both Visual Studio and the SDK, if you haven’t already. The SDK contains files needed for building the library, “Win32.mak” specifically, which doesn’t come with the Express version of Visual Studio.

  2. Unzip the libjpeg library and you’ll end up with a directory path similar to this:
    1
    C:\Users\%USERNAME%\Downloads\lib\jpegsr8d\jpeg-8d\

    Open a console in this directory.

  3. Run Visual Studio’s “vcvarsall.bat” to set the environment variables.
    1
    "C:\Program Files\Microsoft Visual Studio 11.0\VC\vcvarsall.bat"

  4. Set the environment variables to include the “Win32.mak” path.
    1
    set INCLUDE=C:\Program Files (x86)\Microsoft SDKs\Windows\v7.0A\Include;%INCLUDE%

  5. Create the MSVC 2010 solution files using nmake:
    1
    nmake /f makefile.vc setup-v10

Now there should be two newly created Visual Studio solution files: “jpeg.sln” and “apps.sln.” The first is for the jpeg library itself, while the latter is for building a set of applications that utilize the library.

Building

  1. Open the “jpeg.sln” file in MSVC 2012 and allow the project upgrade. The “Release” configuration is the only option.

  2. Set the code generation flag in the project properties for a static (/MT) build.

  3. Build the solution.

  4. The new static library file is created in a new subfolder called “Release.”
    1
    C:\Users\%USERNAME%\Downloads\lib\jpegsr8d\jpeg-8d\Release\jpeg.lib

Installing

  1. Create a place for the jpeg library with “jpeg” and “lib” subfolders.
    1
    2
    mkdir "C:\workspace\lib\jpeg\jpeg-8d\jpeg"
    mkdir "C:\workspace\lib\jpeg\jpeg-8d\lib"

  2. Copy the header files.
    1
    xcopy "C:\Users\%USERNAME%\Downloads\lib\jpegsr8d\jpeg-8d\*.h" "C:\workspace\lib\jpeg\jpeg-8d\jpeg\"

    .

  3. Copy the library file.
    1
    xcopy "C:\Users\%USERNAME%\Downloads\lib\jpegsr8d\jpeg-8d\Release\jpeg.lib" "C:\workspace\lib\jpeg\jpeg-8d\lib\"

  4. Add the include and lib paths to the default project property page in MSVC 2012:
    View → Other Windows → Property Manager → Release/Debug → Microsoft.Cpp.Win32.user    .
    Be sure to save the property sheet so that the changes take effect.

Testing

  1. Create a new project, “LibTest” in MSVC 2012.

  2. Explicitly add the jpeg library to the project: Project → Properties →Linker → Input → Additional Dependencies = “jpeg.lib;”

  3. Create a source file in the project with this code:
    1
    2
    3
    4
    5
    6
    7
    #include <stdio.h>
    #include <jpeglib.h>
    void main()
    {
        j_compress_ptr cinfo;
        jpeg_abort_compress(cinfo);
    }

Build the project. It should compile and link successfully.

Potential Issues

These are some of the problems that you might run into while trying to build libjpeg.

makefile.vc(11) : fatal error U1052: file ‘win32.mak’ not found

The Win32.mak file can’t be located by Visual Studio. It isn’t included with the Express edition, but can be obtained by installing the MS Windows 7 SDK noted above. The file’s location depends on whether you are on a 32-bit or 64-bit system.

32-bit:

1
2
C:\Program Files\Microsoft SDKs\Windows\v7.0\Include
C:\Program Files\Microsoft SDKs\Windows\v7.0A\Include

64-bit:

1
2
C:\Program Files (x86)\Microsoft SDKs\Windows\v7.0A\Include
C:\Program Files (x86)\Microsoft SDKs\Windows\v7.0\Include

Set the “INCLUDE” environment variable to the path with “Win32.mak” in the console:

1
set INCLUDE=C:\Program Files (x86)\Microsoft SDKs\Windows\v7.0A\Include;%INCLUDE%

Verify that the include path is set properly

1
2
echo %INCLUDE%
C:\Program Files (x86)\Microsoft Visual Studio 11.0\VC\INCLUDE;C:\Program Files (x86)\Windows Kits\8.0\include\shared;C:\Program Files (x86)\Windows Kits\8.0\include\um;C:\Program Files (x86)\Windows Kits\8.0\include\winrt;

LNK2001: unresolved external symbol _jpeg_abort_compress

The library file hasn’t been explicitly added to the project. Verify that the “jpeg.lib” file is included in the project’s “Additional Dependencies.”

C2061: syntax error : identifier ‘FILE’

The project can’t find any reference for the structure “FILE.” It is a member of “stdio.h,” which needs to be included before “jpeg.h.”
